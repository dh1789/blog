## 블로그 포스트 요구사항

**주제**: {TOPIC}
**키워드**: {KEYWORDS}
**타겟 단어 수**: {WORDS} 단어
**언어**: {LANGUAGE}

## 타겟 독자

개발자, 기술 실무자, 엔지니어, 그리고 해당 주제에 관심 있는 일반 독자

## 작성 스타일

- **톤**: 친근하면서도 전문적
- **접근법**: 실용적이고 실행 가능한 정보 제공
- **구조**: 논리적이고 체계적인 흐름

## 블로그 포스트 구조

1. **서론** (10%)
   - 독자의 관심을 끄는 흥미로운 시작
   - 주제 소개 및 배경
   - 이 글을 읽어야 하는 이유
   - 독자가 얻을 수 있는 것

2. **핵심 내용** (70%)

   **주요 섹션 1**
   - 핵심 개념 설명
   - 구체적인 예시 및 코드
   - 실제 적용 방법

   **주요 섹션 2**
   - 심화 내용
   - 베스트 프랙티스
   - 일반적인 실수 및 해결책

   **주요 섹션 3**
   - 고급 기법
   - 성능 최적화
   - 실전 팁

3. **결론** (10%)
   - 핵심 내용 요약
   - 실행 가능한 다음 단계
   - 추가 학습 리소스
   - 독자에게 전하는 메시지

4. **추가 정보** (10%)
   - 참고 자료 및 링크
   - 관련 주제 제안
   - FAQ (선택사항)

## 작성 가이드라인

### 필수 포함 요소
- 명확한 제목과 부제목 (H1, H2, H3 마크다운 형식)
- 프론트매터(frontmatter)에 메타데이터 포함:
  ```yaml
  ---
  title: "매력적인 제목: 구체적이고 명확하게"
  excerpt: "핵심 내용 요약 (150자 이내)"
  categories: ["카테고리1", "카테고리2"]
  tags: ["태그1", "태그2", "태그3"]
  date: "YYYY-MM-DD"
  ---
  ```
- 실용적이고 실행 가능한 정보
- 코드 예제 (해당되는 경우)
- 시각적 설명 (다이어그램, 스크린샷 위치 명시)

### 콘텐츠 품질

**명확성**
- 복잡한 개념을 쉽게 설명
- 전문 용어 사용 시 간단히 정의
- 단계별 설명으로 이해 돕기

**실용성**
- 실제로 적용 가능한 정보
- 구체적인 예시와 코드
- 독자가 바로 시도할 수 있는 내용

**정확성**
- 검증된 정보만 포함
- 출처가 명확한 데이터 사용
- 최신 정보 반영

### SEO 최적화

- 타겟 키워드를 자연스럽게 포함
- 제목과 부제목에 키워드 활용
- 메타 설명(excerpt)에 키워드 포함
- 내부 링크 및 외부 링크 적절히 사용

### 가독성 향상

- 짧은 문단 (2-3문장)
- 불릿 포인트 및 번호 목록 활용
- 코드 블록은 언어 명시
- 적절한 헤딩 계층 구조

## 특별 요소

```markdown
## 📌 핵심 요약

이 글에서 다루는 내용:
- 핵심 포인트 1
- 핵심 포인트 2
- 핵심 포인트 3
```

### 팁 박스
```markdown
> 💡 **팁**: 유용한 추가 정보나 베스트 프랙티스
```

### 주의사항 박스
```markdown
> ⚠️ **주의**: 흔한 실수나 주의할 점
```

### 코드 예제
```markdown

```
```

## 결과물 형식

마크다운 형식으로 작성하되, 다음 규칙을 준수:
- 프론트매터를 가장 위에 배치
- 명확한 헤딩 계층 구조 유지 (H1 한 번, H2-H3 활용)
- 코드 블록은 언어 명시 (```javascript, ```python 등)
- 이미지나 다이어그램 위치를 `[이미지: 설명]` 형식으로 명시
- 링크는 신뢰할 수 있는 소스로 (공식 문서, GitHub 등)
- 적절한 줄바꿈으로 가독성 확보
