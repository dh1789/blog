## ✍️ 작성 중 체크

### Frontmatter (메타데이터)
- [ ] **title**: 명확하고 SEO 친화적 (50-60자)
- [ ] **excerpt**: 요약문 작성 (150-160자, SEO 중요)
- [ ] **categories**: 2-3개 (기존 카테고리 우선 사용)
- [ ] **tags**: 5-7개 (핵심 키워드 포함)
- [ ] **status**: `publish` 또는 `draft`
- [ ] **language**: `ko` 또는 `en`

### 구조 및 형식
- [ ] **서론**: 명확한 문제 제기 및 목표 설명 (150-200단어)
- [ ] **본문**: H2 헤더 3개 이상으로 구조화
- [ ] **결론**: 핵심 내용 요약 및 다음 단계 제시
- [ ] **문장 길이**: 평균 20-25단어 (가독성)
- [ ] **문단 길이**: 3-5문장 (모바일 최적화)

### 코드 예제
- [ ] **언어 태그 필수**: ```bash, ```typescript, ```javascript 등
- [ ] **주석 포함**: 코드 이해를 돕는 간결한 주석
- [ ] **실제 동작**: 복사-붙여넣기로 즉시 실행 가능
- [ ] **에러 처리**: try-catch 또는 에러 핸들링 포함

**좋은 예**:
```typescript
// WordPress 클라이언트 생성
const client = new WordPressClient({
  url: process.env.WORDPRESS_URL,
  username: process.env.WORDPRESS_USERNAME,
  password: process.env.WORDPRESS_APP_PASSWORD,
});
```

**나쁜 예**:
```
const client = new WordPressClient(config);
```

### 콘텐츠 품질
- [ ] **독창성**: 실제 경험, 독자적 통찰력, 차별화된 관점
- [ ] **정확성**: 검증된 정보, 공식 문서 참조
- [ ] **실용성**: 즉시 적용 가능한 팁, 단계별 가이드
- [ ] **최신성**: 2025년 기준 최신 정보
- [ ] **완성도**: 도입-전개-결론 명확

### 이미지 및 미디어 (선택)
- [ ] 스크린샷 또는 다이어그램 추가
- [ ] 이미지 alt 태그 작성 (SEO)
- [ ] 이미지 최적화 (WebP, <200KB)

## 🔍 작성 후 검토

### SEO 최적화
- [ ] **키워드 밀도**: Primary 키워드 0.5-2.5%
- [ ] **제목에 키워드 포함**: 자연스럽게 배치
- [ ] **H2 헤더에 키워드 포함**: 1-2개 헤더
- [ ] **첫 문단에 키워드 등장**: 100단어 이내
- [ ] **내부 링크**: 기존 포스트 2-3개 링크
- [ ] **외부 링크**: 신뢰할 수 있는 출처 2-3개

### 가독성
- [ ] 짧은 문장 및 문단
- [ ] 불릿 리스트 활용 (3개 이상 항목)
- [ ] 번호 리스트 활용 (단계별 가이드)
- [ ] 강조 표시: **굵게**, *이탤릭*, `코드`
- [ ] 인용구 또는 팁 박스 (선택)

### 기술 검증
- [ ] **코드 실행 테스트**: 모든 예제 실제 동작 확인
- [ ] **링크 확인**: 깨진 링크 없음
- [ ] **맞춤법 검사**: 한국어/영어 맞춤법
- [ ] **용어 일관성**: 같은 개념은 동일한 용어 사용

## 🚀 발행 전 최종 체크

### CLI 명령어 실행
```bash
# SEO 점수 확인 (자동)
node packages/cli/dist/index.mjs publish "content/posts/ko/your-post.md" --dry-run

# 최소 목표: 60점 이상
# 권장: 70점 이상
```

### 최종 검토 항목
- [ ] **SEO 점수**: 60점 이상 확인
- [ ] **키워드 경고**: 중요 경고 해결
- [ ] **최소 길이**: 짧은 포스트 500단어, 긴 포스트 1500단어
- [ ] **코드 블록**: 모든 언어 태그 확인
- [ ] **frontmatter**: 모든 필드 완성

### 발행 옵션
```bash
# 초안으로 발행 (비공개)
blog publish "content/posts/ko/your-post.md" --draft

# 즉시 발행 (공개)
blog publish "content/posts/ko/your-post.md"
```
