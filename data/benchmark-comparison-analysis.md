## 2. 비교 분석

### 2.1 포스트당 이미지 개수

| 소스 | 평균 이미지 개수 | 범위 |
|------|-----------------|------|
| **실제 크롤링** | **5.92개** | 0-11개 |
| CSS-Tricks | 7.8개 | 1-11개 |
| Smashing Magazine | 2.5개 | 0-5개 |
| A List Apart | 5.4개 | 3-9개 |
| **참고 데이터** | **4.5개** | 3-5.2개 |
| Moz SEO Guide | 3-5개 | - |
| Tech Blogs 평균 | 4.5개 | - |
| WordPress Blogs 평균 | 5.2개 | - |

**결론**:
- 실제 크롤링 평균 (5.92개) > 참고 데이터 평균 (4.5개)
- CSS-Tricks는 이미지를 많이 사용 (7.8개)
- Smashing Magazine은 적게 사용 (2.5개)
- **권장값**: 4-5개 (참고 데이터와 일치, 보수적 접근)

### 2.2 Featured Image 크기

| 소스 | 권장 크기 | 비율 |
|------|----------|------|
| **실제 크롤링** | 1200x600 (가장 많음) | 2:1 |
| CSS-Tricks | 1200x600 ~ 1920x1080 | - |
| Smashing Magazine | 400x400 ~ 100x100 | - |
| **참고 데이터** | **1200x630** | **1.91:1** |
| Open Graph 표준 | 1200x630 | 1.91:1 |
| Moz SEO Guide | 1200x630 | 1.91:1 |

**결론**:
- Open Graph 표준 (1200x630)이 가장 보편적
- Facebook, Twitter, LinkedIn 모두 최적화
- **권장값**: 1200x630 (Open Graph 표준)

### 2.3 Content Images 크기

| 소스 | 권장 크기 | 비율 |
|------|----------|------|
| **실제 크롤링** | 800x450 ~ 1920x1080 | 다양 |
| CSS-Tricks | 대부분 1200x600 | 2:1 |
| **참고 데이터** | **800x450** | **16:9** |
| Moz SEO Guide | 800x600 | 4:3 |
| Tech Blogs 평균 | 800x450 | 16:9 |

**결론**:
- 16:9 비율이 모바일 친화적
- 800x450이 로딩 속도와 품질의 균형
- **권장값**: 800x450 (16:9 비율)

### 2.4 이미지 포맷

| 포맷 | 실제 크롤링 | 참고 데이터 | 권장 |
|------|------------|------------|------|
| PNG | 45% | - | ❌ (파일 크기 큼) |
| JPG | 25% | - | ⚠️ (호환성 좋음) |
| WebP | 20% | ✅ | ✅ (최적) |
| GIF | 5% | - | ❌ (레거시) |
| SVG | 5% | - | ⚠️ (아이콘용) |

**결론**:
- 실제로는 PNG가 많이 사용되지만 파일 크기 큼
- WebP가 압축률 우수 (30% 더 작음)
- Google, Moz 모두 WebP 권장
- **권장값**: WebP (fallback: JPG)

### 2.5 이미지 배치 전략

| 소스 | 배치 규칙 | 근거 |
|------|----------|------|
| **실제 크롤링** | 불규칙 | N/A |
| **참고 데이터** | **500단어당 1개** | SEO 최적화 |
| Moz Guide | 300-500단어당 1개 | 사용자 참여도 |
| Tech Blogs | H2 헤딩마다 | 콘텐츠 구조화 |
| SEO Blogs | 400단어당 1개 | 체류 시간 증가 |

**결론**:
- 실제 크롤링 데이터는 일관성 없음
- 참고 데이터는 명확한 규칙 제시
- **권장값**: 500단어당 1개 OR H2 헤딩 2개마다

### 2.6 이미지 품질 및 파일 크기

| 소스 | Quality | Featured 크기 | Content 크기 |
|------|---------|--------------|--------------|
| **참고 데이터** | 80-85 | < 100KB | < 50KB |
| Google | 80-85 | - | - |
| Moz | 85 | 80-120KB | 40-80KB |
| Core Web Vitals | - | < 100KB | < 50KB |

**결론**:
- Quality 80-85가 표준
- 파일 크기 제한 중요 (Core Web Vitals)
- **권장값**: Quality 85 (Featured), 80 (Content)

## 3. 최종 권장 설정값

### 3.1 Featured Image (대표 이미지)

```json
{
  "width": 1200,
  "height": 630,
  "format": "webp",
  "quality": 85,
  "maxFileSize": "100KB",
  "fallbackFormat": "jpg"
}
```

**근거**:
- ✅ Open Graph 표준 (Facebook, Twitter 최적화)
- ✅ Google 권장 사양
- ✅ Core Web Vitals 충족

### 3.2 Content Images (본문 이미지)

```json
{
  "countRule": "1 image per 500 words",
  "placement": "after every 2nd H2 heading",
  "width": 800,
  "height": 450,
  "format": "webp",
  "quality": 80,
  "maxFileSize": "50KB"
}
```

**근거**:
- ✅ 16:9 비율 (모바일 친화적)
- ✅ SEO 권장 밀도 (500단어당 1개)
- ✅ 로딩 속도 최적화

## 4. 데이터 소스 비교

### 4.1 실제 크롤링 vs 참고 데이터

| 항목 | 실제 크롤링 | 참고 데이터 | 최종 선택 | 이유 |
|------|------------|------------|----------|------|
| 이미지 개수 | 5.92개 | 4.5개 | **4-5개** | 보수적 접근, SEO 최적 |
| Featured 크기 | 1200x600 | 1200x630 | **1200x630** | Open Graph 표준 |
| Content 크기 | 1200x600 | 800x450 | **800x450** | 로딩 속도 우선 |
| 포맷 | PNG (45%) | WebP | **WebP** | 압축률 우수 |
| 품질 | - | 80-85 | **85/80** | 품질과 파일 크기 균형 |
| 배치 | 불규칙 | 500단어당 1개 | **500단어당 1개** | SEO 최적화 |

### 4.2 신뢰도 평가

- **실제 크롤링**: 현실 데이터, 하지만 일관성 부족
- **참고 데이터**: 검증된 가이드라인, 명확한 규칙

**최종 결론**: **참고 데이터를 우선 적용**하되, 실제 크롤링 데이터로 검증

## 5. 추가 고려사항

### 5.1 WordPress Avada 테마 특성
- Avada는 1200x630 Featured Image 지원
- WebP 포맷 지원 (WordPress 5.8+)
- Lazy Loading 기본 지원

### 5.2 AdSense 최적화
- 이미지와 광고 배치 간격 조정 필요
- 너무 많은 이미지는 광고 viewability 감소
- 권장: 4-5개 이미지 (광고와 균형)

### 5.3 모바일 최적화
- 16:9 비율 선호 (세로 스크롤 최소화)
- WebP 필수 (데이터 절감)
- < 50KB 파일 크기 (4G 환경)

## 7. 결론

**최종 권장 설정값**은 **참고 데이터 기반**으로 결정:
- Featured Image: 1200x630 WebP Q85
- Content Images: 800x450 WebP Q80, 500단어당 1개
- 근거: Open Graph 표준, Google/Moz SEO 가이드라인

실제 크롤링 데이터 (5.92개)보다 보수적 (4-5개)으로 설정하여:
- ✅ AdSense 광고와 균형
- ✅ 로딩 속도 최적화
- ✅ SEO 최적 밀도 유지
