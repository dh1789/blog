## 타겟 독자

AI 도구와 최신 기술을 배우고 싶은 일반 사용자 및 개발자

## 작성 스타일

- **톤**: 친근하고 실용적이며 전문적
- **접근법**: 초보자도 이해할 수 있도록 쉽게 설명하되, 기술적 깊이도 제공
- **구조**: 명확한 섹션 구분과 실용적인 예제 포함

## 콘텐츠 구조

### 1. 도입부 (10-15%)
- 주제와 관련된 현실적인 문제나 기회 제시
- 독자가 왜 이 포스트를 읽어야 하는지 명확히 설명
- 핵심 키워드 자연스럽게 포함

### 2. 본문 (70-80%)
- **개념 설명**: 주요 용어와 개념을 명확하게 정의
- **실용적 가이드**: 단계별 설명 또는 방법론 제시
- **실제 예제**: 구체적인 사례나 코드 예제 포함
- **시각 자료**: 필요시 표, 다이어그램, 스크린샷 설명 포함
- **팁과 주의사항**: 실무에서 유용한 조언과 피해야 할 함정 설명

### 3. 결론 (10-15%)
- 핵심 내용 요약
- 독자가 실천할 수 있는 명확한 다음 단계 제시
- 관련 리소스나 추가 학습 자료 추천

## 작성 가이드라인

### 기술 용어 사용
- 전문 용어 사용 시 반드시 설명 포함
- 영문 용어는 한글 번역과 함께 제공 (예: "API (Application Programming Interface)")
- 코드 예제는 주석으로 상세히 설명

### 예제와 실습
- 가능한 한 실행 가능한 예제 제공
- 예제는 실제 사용 사례를 반영
- 단계별로 따라 할 수 있도록 명확한 지침 제공

### 참고 자료
- 공식 문서, 신뢰할 수 있는 출처 인용
- 외부 링크는 독자에게 실질적인 가치를 제공하는 것만 포함
- 가능하면 최신 정보 사용 (2024-2025년 기준)

## SEO 최적화

- **제목**: 명확하고 검색 의도를 반영한 제목 (50-60자)
- **키워드 사용**: 자연스럽게 본문 전체에 분산
- **부제목**: 명확한 계층 구조 (H2, H3 활용)
- **메타 설명**: 간결하고 매력적인 요약 (150-160자)

## 가독성

- **문단**: 3-4문장으로 짧게 유지
- **목록**: 복잡한 정보는 불릿 포인트나 번호 목록 사용
- **강조**: 중요한 내용은 **굵게** 또는 _기울임꼴_ 사용
- **공백**: 적절한 여백으로 가독성 향상

## 결과물 형식

마크다운 형식으로 작성하되, 다음 요소를 포함:

- 명확한 제목 (H1)
- 논리적인 섹션 구분 (H2, H3)
- 코드 블록 (해당 시)
- 인용구 (중요한 포인트 강조)
- 링크와 참고 자료

다른 설명 없이 블로그 포스트 마크다운만 출력하세요.
